# Technical Summary

This repo is a compact Node/Express app with a static single-page frontend.

## Runtime

- Node.js `>=20`
- Express
- Multer for image uploads
- No database service yet
- JSON file store under `DATA_DIR`
- PM2 in production
- Nginx path proxy under `/Manager_Site/`

## Important Paths

Local:

- `server.js`
- `public/index.html`
- `public/app.js`
- `public/styles.css`
- `scripts/hash-password.js`
- `data/` for local development data

Production:

- `/root/Manager_Site`
- `/root/Manager_Site/.env`
- `/root/Manager_Site/data/store.json`
- `/root/Manager_Site/data/uploads`
- `/root/Manager_Site/data/initial-admin.txt`

## Authentication

The app uses server-side in-memory sessions keyed by an `HttpOnly` cookie:

- Cookie name: `manager_site_session`
- Session TTL: 8 hours
- Password hashing: Node `crypto.scrypt`
- Production first admin password is generated by `deploy_linux.sh`

Because sessions are in memory, users may need to log in again after a PM2 restart. That is acceptable for the current app, but a future persistent session store would improve production behavior.

## Data Model

The JSON store contains:

- `users`
- `sites`
- `audit`

Important user fields:

- `username`
- `displayName`
- `role`
- `active`
- `siteId`
- `permissions`
- `passwordHash`

Important site fields:

- `ownerUsername`
- `name`
- `websiteUrl`
- `status`
- `slots`
- `images`

## API Areas

Auth:

- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/auth/me`

Admin:

- `GET /api/admin/users`
- `POST /api/admin/users`
- `PATCH /api/admin/users/:userId`
- `GET /api/admin/audit`

Sites:

- `GET /api/sites`
- `GET /api/sites/:siteId`
- `PATCH /api/sites/:siteId`
- `POST /api/sites/:siteId/status`
- `POST /api/sites/:siteId/images`
- `DELETE /api/sites/:siteId/images/:imageId`

## Frontend Routing

The SPA routes are:

- `/login`
- `/admin`
- `/client/:username`

Do not change `/client/:username` to a Hebrew backend path unless the user explicitly asks for backend routing changes. Hebrew applies to client-facing UI text and direction.

## Current UI Language Direction

- Login: Hebrew, RTL
- Client workspace: Hebrew, RTL
- Admin workspace: English, LTR
